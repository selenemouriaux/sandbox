Menu responsive

Kesako ?

Un menu responsive consiste à cacher le menu général du site, généralement présenté dans une navigation horizontale, laquelle ne peut être affichée sur une seule ligne lorsque la largeur d'écran est inférieure à un certain seuil (breakpoint)

Sur les mobiles, le menu est souvent dépliable en actionnant un bouton, lequel porte le nom de "Burger" par référence à sa ressemblance à un hamburger :

Éléments fournis

Le présent dossier de départ founit :

  1. Une feuille de style
  2. Un document html
  3. Des liens de ressources dans la navigation
  4. Un fichier javascript de démarrage, introduisant une fonction toggleNav() restant à développer et à exécuter.

La feuille de style

Elle met à votre disposition les règles CSS suivantes :

  1. La classe button-burger, associée à un élement html <button>, permet de dessiner un bouton de style burger.
    Les plus curieux pourront observer le code CSS : le bouton est dessiné en CSS.
  2. La classe has-focus asssociée au burger permet de modifier son apparence : l'élément se présente alors comme une croix (également réalisée en CSS)
  3. L'identifiant #menu, d'ores et déjà appliqué à l'élément <nav> de la bannière, subit une transformation de style pour une largeur d'écran inférieure à 1023px.

Comportement du menu

Lorsque la largeur d'écran est inférieure à 1024px, la navigation n'est plus visible.

Dans ce cas, l'ajout de la classe is-open permet son affichage avec un effet d'animation, exploitant la propriété CSS transform afin de provoquer un glissement depuis la gauche de l'écran

Votre mission

Votre mission, si vous l'acceptez, est de réaliser le fonctionnement javascript suivant :

  1. Le javascript doit être chargé dans le document
  2. Le code ne doit s'exécuter qu'en dessous de 1024px : voir La méthode matchMedia ou les Media Queries pour JavaScript
  3. Création d'un élément <button> affecté de la classe button-burger
  4. Insertion du bouton entre <p class="site-description"> et <nav>
  5. Installation d'un gestionnaire d'événement au click sur le bouton qui doit réaliser :
    1. Un toggle sur le menu de la class is-open
    2. Un toggle sur le bouton de la class has-focus
  6. Le code ne doit s'exécuter qu'une fois les ressources entièrement chargées : (re)voir L’évènement load
  7. En bonus : prévoir de recharger la page si elle est redimmensionnée :
    • Voir : element.onresize
    • Voir : Location : reload()

Test de votre code

Il va s'en dire, mais nous le rappelons quand même, que vous devez tester votre code en utilisant les outils médias du navigateur (simuler l'affichage sur un écran mobile)

De même, gardez un oeil sur la console : elle est votre seule amie !
